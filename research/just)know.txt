# 🔹 A. Missing Value Handling

### Numerical

* Mean
* Median (preferred)
* KNN imputer

```python

```

---

# 🔹 B. Outlier Handling

Options:

### 1️⃣ Capping (Winsorization)

Better than removing rows.

### 2️⃣ Robust Scaling (Recommended)

```python

```

# 🔹 C. Feature Scaling Methods

| ------------------- | ------------------- |

| RobustScaler        | Outliers present    |
| PowerTransformer    | Skewed data         |
| QuantileTransformer | Highly skewed       |

# 🔹 D. Handling Skewness

# 🧠 FULL PROFESSIONAL PIPELINE (BEST PRACTICE)

Instead of doing manually:

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import RobustScaler
from sklearn.impute import SimpleImputer
from sklearn.ensemble import RandomForestClassifier

pipeline = Pipeline([
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", RobustScaler()),
    ("model", RandomForestClassifier())
])

pipeline.fit(X_train, y_train)
```

This prevents leakage automatically.

Handling **imbalanced data** is VERY important — especially for classification problems like wine quality.

---

# 🧠 First — What is Imbalanced Data?

Imbalanced = One class has much more samples than others.

Example:

| Quality | Count |
| ------- | ----- |
| 5       | 680   |
| 6       | 630   |
| 3       | 10    |
| 8       | 18    |

Model will just predict 5 or 6 → high accuracy but poor real performance.

---

# ✅ STEP 1 — Check Class Distribution

Before doing anything:

```python
y.value_counts()
```

Or percentage:

```python
y.value_counts(normalize=True) * 100
```

If one class < 10% → it's imbalanced.

---

# 🚨 VERY IMPORTANT RULE

👉 Handle imbalance **ONLY on training data**
👉 NEVER touch test data

---

# 📌 Complete Imbalance Handling Techniques

There are 3 categories:

---

# 🔹 1️⃣ Data-Level Methods (Resampling)

### A. Oversampling (Increase minority)

## ✅ SMOTE (Most Popular)

```python
from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42)

X_train_resampled, y_train_resampled = smote.fit_resample(X_train, y_train)
```

✔ Creates synthetic samples
✔ Better than simple duplication

---

### B. Random Oversampling

```python
from imblearn.over_sampling import RandomOverSampler

ros = RandomOverSampler(random_state=42)
X_train_resampled, y_train_resampled = ros.fit_resample(X_train, y_train)
```

⚠️ Can cause overfitting

---

### C. Undersampling (Reduce majority)

```python
from imblearn.under_sampling import RandomUnderSampler

rus = RandomUnderSampler(random_state=42)
X_train_resampled, y_train_resampled = rus.fit_resample(X_train, y_train)
```

⚠️ Risk of losing information

---

# 🔹 2️⃣ Algorithm-Level Methods

Instead of changing data, change model.

## ✅ Use class_weight

For Logistic Regression / RandomForest:

```python
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(class_weight="balanced")
```

OR custom weights:

```python
model = RandomForestClassifier(class_weight={0:1, 1:5})
```

✔ Very clean solution
✔ No synthetic data

---

# 🔹 3️⃣ Evaluation-Level Methods

Never rely on accuracy.

Use:

```python
from sklearn.metrics import classification_report
```

Important metrics:

| Metric           | Why                        |
| ---------------- | -------------------------- |
| Precision        | Avoid false positives      |
| Recall           | Avoid false negatives      |
| F1-score         | Balance precision & recall |
| ROC-AUC          | Overall performance        |
| Confusion Matrix | Visual error               |

# 🧠 Professional Pipeline with SMOTE

```python
from imblearn.pipeline import Pipeline
from imblearn.over_sampling import SMOTE
from sklearn.preprocessing import RobustScaler
from sklearn.impute import SimpleImputer
from sklearn.ensemble import RandomForestClassifier

pipeline = Pipeline([
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", RobustScaler()),
    ("smote", SMOTE(random_state=42)),
    ("model", RandomForestClassifier())
])

pipeline.fit(X_train, y_train)
```

This avoids leakage.
